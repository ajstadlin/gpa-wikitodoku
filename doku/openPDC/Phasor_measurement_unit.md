            <h1 id="firstHeading" class="firstHeading" lang="en">Phasor measurement unit</h1>
                                    <div id="bodyContent" class="mw-body-content">
                                    <div id="siteSub">From Wikipedia, the free encyclopedia</div>
                                <div id="contentSub"></div>
                                                <div id="jump-to-nav" class="mw-jump">

Using a PMU, it is simple to detect abnormal waveform shapes. A waveform shape described mathematically is called a phasor.</div>

<p>A <b>phasor measurement unit</b> (PMU) or <b>synchrophasor</b> is a device which measures the electrical <a href="/wiki/Waveform" title="Waveform">waves</a> on an <a href="/wiki/Grid_(electricity)" title="Grid (electricity)" class="mw-redirect">electricity grid</a>, using a common time source for synchronization. Time synchronization allows synchronized real-time measurements of multiple remote measurement points on the grid. In <a href="/wiki/Power_engineering" title="Power engineering">power engineering</a>, these are also commonly referred to as synchrophasors and are considered one of the most important measuring devices in the future of power systems.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span>[</span>1<span>]</span></a></sup> A PMU can be a dedicated device, or the PMU function can be incorporated into a <a href="/wiki/Protective_relay" title="Protective relay">protective relay</a> or other device.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span>[</span>2<span>]</span></a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 1893, <a href="/wiki/Charles_Proteus_Steinmetz" title="Charles Proteus Steinmetz">Charles Proteus Steinmetz</a> presented a paper on simplified mathematical description of the waveforms of alternating current electricity. Steinmetz called his representation a <a href="/wiki/Phasor_(sine_waves)" title="Phasor (sine waves)" class="mw-redirect">phasor</a>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>3<span>]</span></a></sup> With the invention of phasor measurement units (PMU) in 1988 by Dr. Arun G. Phadke and Dr. James S. Thorp at Virginia Tech, Steinmetz’s technique of phasor calculation evolved into the calculation of real time phasor measurements that are synchronized to an absolute time reference provided by the <a href="/wiki/Global_Positioning_System" title="Global Positioning System">Global Positioning System</a>. Early prototypes of the PMU were built at <a href="/wiki/Virginia_Polytechnic_Institute_and_State_University" title="Virginia Polytechnic Institute and State University" class="mw-redirect">Virginia Tech</a>, and <a rel="nofollow" class="external text" href="http://www.macrodyneusa.com/model_1690.htm">Macrodyne</a> built the first PMU (model 1690) in 1992.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Operation">Operation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=2" title="Edit section: Operation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A PMU can measure 50/60&#160;Hz AC waveforms (voltages and currents) typically at a rate of 48 samples per cycle (2880 samples per second for 60Hz systems). The analog AC waveforms are digitized by an <a href="/wiki/Analog_to_Digital_converter" title="Analog to Digital converter" class="mw-redirect">Analog to Digital converter</a> for each phase. A phase-lock oscillator along with a <a href="/wiki/Global_Positioning_System" title="Global Positioning System">Global Positioning System</a> (GPS) reference source provides the needed high-speed synchronized sampling with 1 microsecond accuracy. The resultant time tagged phasors can be transmitted to a local or remote receiver at rates up to 60 samples per second.<sup id="cite_ref-CERTS_5-0" class="reference"><a href="#cite_note-CERTS-5"><span>[</span>5<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Technical_overview">Technical overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=3" title="Edit section: Technical overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A <a href="/wiki/Phasor_(sine_waves)" title="Phasor (sine waves)" class="mw-redirect">phasor</a> is a complex number that represents both the magnitude and phase angle of the sine waves found in electricity. Phasor measurements that occur at the same time are called "synchrophasors", as are the PMU devices that allow their measurement. In typical applications phasor measurement units are sampled from widely dispersed locations in the power system network and synchronized from the common time source of a <a href="/wiki/Global_positioning_system" title="Global positioning system" class="mw-redirect">global positioning system (GPS)</a> <a href="/wiki/Radio_clock" title="Radio clock">radio clock</a>. Synchrophasor technology provides a tool for system operators and planners to measure the state of the electrical system and manage <a href="/wiki/Power_quality" title="Power quality">power quality</a>.</p>
<p>Synchrophasors measure voltages and currents at principal intersecting locations (critical substations) on a power grid and can output accurately time-stamped voltage and current phasors. Because these phasors are truly synchronized, synchronized comparison of two quantities is possible in real time. These comparisons can be used to assess system conditions-such as; frequency changes, MW, MVARs, kVolts, etc.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (April 2015)">clarification needed</span></a></i>]</sup> The monitored points are preselected through various studies to make extremely accurate phase angle measurements to indicate shifts in system (grid) stability. The phasor data is collected either on-site or at centralized locations using Phasor Data Concentrator technologies. The data is then transmitted to a regional monitoring system which is maintained by the local Independent System Operator (ISO). These ISO's will monitor phasor data from individual PMU's or from as many as 150 PMU's - this monitoring provides an accurate means of establishing controls for power flow from multiple energy generation sources (nuclear, coal, wind, etc.).</p>
<p>The technology has the potential to change the economics of power delivery by allowing increased power flow over existing lines. Synchrophasor data could be used to allow power flow up to a line's dynamic limit instead of to its worst-case limit.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (April 2015)">clarification needed</span></a></i>]</sup> Synchrophasor technology will usher in a new process for establishing centralized and selective controls for the flow of electrical energy over the grid. These controls will affect both large scale (multiple-states) and individual transmission line sections at intersecting substations. Transmission line congestion (over-loading) and protection and controls will therefore be improved on a multiple region scale (US, Canada, Mexico) through interconnecting ISO's.</p>
<h2><span class="mw-headline" id="Phasor_networks">Phasor networks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=4" title="Edit section: Phasor networks">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A phasor network consists of phasor measurement units (PMUs) dispersed throughout the electricity system, <a href="/w/index.php?title=Phasor_Data_Concentrator&amp;action=edit&amp;redlink=1" class="new" title="Phasor Data Concentrator (page does not exist)">Phasor Data Concentrators (PDC)</a> to collect the information and a <a href="/wiki/SCADA" title="SCADA">Supervisory Control And Data Acquisition</a> (SCADA) system at the central control facility. Such a network is used in Wide Area Measurement Systems (WAMS), the first of which was begun in 2000 by the <a href="/wiki/Bonneville_Power_Administration" title="Bonneville Power Administration">Bonneville Power Administration</a>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span>[</span>6<span>]</span></a></sup> The complete network requires rapid data transfer within the frequency of sampling of the phasor data. GPS time stamping can provide a theoretical accuracy of synchronization better than 1 <a href="/wiki/Microsecond" title="Microsecond">microsecond</a>. “Clocks need to be accurate to ± 500 nanoseconds to provide the one microsecond time standard needed by each device performing synchrophasor measurement.” <sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span>[</span>7<span>]</span></a></sup> For 60&#160;Hz systems, PMUs must deliver between 10 and 30 synchronous reports per second depending on the application. The PDC correlates the data, and controls and monitors the PMUs (from a dozen up to 60).<sup id="cite_ref-WAMS-1_8-0" class="reference"><a href="#cite_note-WAMS-1-8"><span>[</span>8<span>]</span></a></sup> At the central control facility, the SCADA system presents system wide data on all <a href="/wiki/Electrical_generator" title="Electrical generator" class="mw-redirect">generators</a> and <a href="/wiki/Electrical_substation" title="Electrical substation">substations</a> in the system every 2 to 10 seconds.</p>
<p>PMUs often use phone lines to connect to PDCs, which then send data to the SCADA or Wide Area Measurement System (WAMS) server.<sup id="cite_ref-ESPRI-1_9-0" class="reference"><a href="#cite_note-ESPRI-1-9"><span>[</span>9<span>]</span></a></sup> Additionally, PMUs can use ubiquitous <a href="/wiki/Cellular_network" title="Cellular network">mobile (cellular) networks</a> for data transfer (<a href="/wiki/GPRS" title="GPRS" class="mw-redirect">GPRS</a>, <a href="/wiki/UMTS" title="UMTS" class="mw-redirect">UMTS</a>), which allows potential savings in infrastructure and deployment costs, at the expense of a larger <a href="/wiki/Data_reporting" title="Data reporting">data reporting</a> latency.<sup id="cite_ref-SKOK-1_10-0" class="reference"><a href="#cite_note-SKOK-1-10"><span>[</span>10<span>]</span></a></sup><sup id="cite_ref-WAMSTER-1_11-0" class="reference"><a href="#cite_note-WAMSTER-1-11"><span>[</span>11<span>]</span></a></sup> However, the introduced data latency makes such systems more suitable for R&amp;D measurement campaigns and <a href="/wiki/Real-time_computing#Near_real-time" title="Real-time computing">near real-time</a> monitoring, and limits their use in real-time protective systems.</p>
<p>PMUs from multiple vendors can yield inaccurate readings. In one test, readings differed by 47 microseconds – or a difference of 1 degree of at 60&#160;Hz- an unacceptable variance.<sup id="cite_ref-NASPI-1_12-0" class="reference"><a href="#cite_note-NASPI-1-12"><span>[</span>12<span>]</span></a></sup> China's solution to the problem was to build all its own PMUs adhering to its own specifications and standards so there would be no multi-vendor source of conflicts, standards, protocols, or performance characteristics.<sup id="cite_ref-Yang_13-0" class="reference"><a href="#cite_note-Yang-13"><span>[</span>13<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Installation">Installation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=5" title="Edit section: Installation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Installation of a typical 10 Phasor PMU is a simple process. A phasor will be either a 3 phase voltage or a 3 phase current. Each phasor will, therefore, require 3 separate electrical connections (one for each phase). Typically an electrical engineer designs the installation and interconnection of a PMU at a substation or at a generation plant. Substation personnel will bolt equipment rack to the floor of the substation following established seismic mounting requirements. Then the PMU along with a modem and other support equipment will be mounted on the equipment rack. They will also install the Global Positioning Satellite (GPS) antenna on the roof of the substation per manufacturer instructions. Substation personnel will also install "shunts" in all <a href="/wiki/Current_transformer" title="Current transformer">Current transformer</a> (CT) secondary circuits that are to be measured. The PMU will also require communication circuit connection (<a href="/wiki/Modem" title="Modem">Modem</a> if using 4-wire connection or <a href="/wiki/Ethernet" title="Ethernet">Ethernet</a> for network connection).<sup id="cite_ref-CERTS_5-1" class="reference"><a href="#cite_note-CERTS-5"><span>[</span>5<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=6" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>The <a href="/wiki/Bonneville_Power_Administration" title="Bonneville Power Administration">Bonneville Power Administration</a> (BPA) is the first utility to implement comprehensive adoption of synchrophasors in its wide-area monitoring system. Today there are several implementations underway.</li>
<li>The <a href="/wiki/FNET" title="FNET">FNET</a> project operated by Virginia Tech and the University of Tennessee utilizes a network of approximately 80 low-cost, high-precision Frequency Disturbance Recorders to collect syncrophasor data from the U.S. power grid. <a rel="nofollow" class="external autonumber" href="http://powerit.utk.edu/fnet.html">[1]</a></li>
<li>The <a href="/wiki/NYISO#NYISO" title="NYISO" class="mw-redirect">New York Independent System Operator</a> has installed 48 PMUs throughout New York State, partly in response to a devastating 2003 blackout that originated in Ohio and affected regions in both the <a href="/wiki/United_States" title="United States">United States</a> and <a href="/wiki/Canada" title="Canada">Canada</a>.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span>[</span>14<span>]</span></a></sup></li>
<li>In 2006, China's Wide Area Monitoring Systems (WAMS) for its 6 grids had 300 PMUs installed mainly at 500 kV and 330 kV substations and power plants. By 2012, China plans to have PMUs at all 500kV substations and all powerplants of 300MW and above. Since 2002, China has built its own PMUs to its own national standard. One type has higher sampling rates than typical and is used in power plants to measure rotor angle of the generator, reporting excitation voltage, excitation current, valve position, and output of the power system stabilizer (PSS). All PMUs are connected via private network, and samples are received within 40 ms on average.<sup id="cite_ref-Yang_13-1" class="reference"><a href="#cite_note-Yang-13"><span>[</span>13<span>]</span></a></sup></li>
<li>The <a rel="nofollow" class="external text" href="http://www.naspi.org">North American Synchrophasor Initiative</a> (NASPI), previously known as The Eastern Interconnect Phasor Project (EIPP), has over 120 connected phasor measurement units collecting data into a "Super Phasor Data Concentrator" system centered at <a href="/wiki/Tennessee_Valley_Authority" title="Tennessee Valley Authority">Tennessee Valley Authority</a> (TVA). This data concentration system is now an open source project known as the <a href="/wiki/OpenPDC" title="OpenPDC">openPDC</a>.</li>
<li>The <a href="/wiki/United_States_Department_of_Energy" title="United States Department of Energy">DOE</a> has sponsored several related research projects, including GridStat <a rel="nofollow" class="external autonumber" href="http://www.gridstat.net">[2]</a> at <a href="/wiki/Washington_State_University" title="Washington State University">Washington State University</a>.</li>
</ul>
<h2><span class="mw-headline" id="Applications">Applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=7" title="Edit section: Applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ol>
<li><a href="/wiki/Power_system_automation" title="Power system automation" class="mw-redirect">Power system automation</a>, as in <a href="/wiki/Smart_grid" title="Smart grid">smart grids</a></li>
<li><a href="/wiki/Load_shedding" title="Load shedding" class="mw-redirect">Load shedding</a> and other <a href="/wiki/Load_control" title="Load control" class="mw-redirect">load control</a> techniques such as <a href="/wiki/Demand_response" title="Demand response">demand response</a> mechanisms to manage a power system. (i.e. Directing power where it is needed in real-time)</li>
<li>Increase the reliability of the power grid by detecting faults early, allowing for isolation of operative system, and the prevention of <a href="/wiki/Power_outage" title="Power outage">power outages</a>.</li>
<li>Increase <a href="/wiki/Power_quality" title="Power quality">power quality</a> by precise analysis and <a href="/wiki/Power_system_automation" title="Power system automation" class="mw-redirect">automated correction</a> of sources of system degradation.</li>
<li>Wide area measurement and control through state estimation,<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span>[</span>15<span>]</span></a></sup> in very wide area <a href="/wiki/Grid_(electricity)#Super_grid" title="Grid (electricity)" class="mw-redirect">super grids</a>, regional <a href="/wiki/Electric_power_transmission" title="Electric power transmission">transmission networks</a>, and <a href="/wiki/Electric_distribution_network" title="Electric distribution network">local distribution</a> grids.</li>
<li>Phasor measurement technology and synchronized time stamping can be used for Security improvement through synchronized encryptions like trusted sensing base. <sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span>[</span>16<span>]</span></a></sup></li>
</ol>
<h2><span class="mw-headline" id="Standards">Standards</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Phasor_measurement_unit&amp;action=edit&amp;section=8" title="Edit section: Standards">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a href="/wiki/IEEE_1344" title="IEEE 1344">IEEE 1344</a> standard for synchrophasors was completed in 1995, and reaffirmed in 2001. In 2005, it was replaced by <a href="/w/index.php?title=IEEE_C37.118-2005&amp;action=edit&amp;redlink=1" class="new" title="IEEE C37.118-2005 (page does not exist)">IEEE C37.118-2005</a>, which was a complete revision and dealt with issues concerning use of PMUs in electric power systems. The specification describes standards for measurement, the method of quantifying the measurements, testing &amp; certification requirements for verifying accuracy, and data transmission format and protocol for real-time data communication.<sup id="cite_ref-ESPRI-1_9-1" class="reference"><a href="#cite_note-ESPRI-1-9"><span>[</span>9<span>]</span></a></sup> This standard was not comprehensive- it did not attempt to address all factors that PMUs can detect in power system dynamic activity.<sup id="cite_ref-WAMS-1_8-1" class="reference"><a href="#cite_note-WAMS-1-8"><span>[</span>8<span>]</span></a></sup> A new version of the standard was released in December 2011, which split the IEEE C37.118-2005 standard into two parts: C37.118-1 dealing with the phasor estimation &amp; C37.118-2 the communications protocol. It also introduced two classifications of PMU, M - measurement &amp; P - protection. M class is close in performance requirements to that in the original 2005 standard, primarily for steady state measurement. P class has relaxed some performance requirements and is intended to capture dynamic system behaviour.</p>
<p>Other standards used with PMU interfacing:</p>
<ul>
<li><a href="/wiki/OLE_for_process_control" title="OLE for process control" class="mw-redirect">OPC-DA / OPC-HDA</a> - A <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> based interface protocol that is currently being generalized to use <a href="/wiki/XML" title="XML">XML</a> and run on non Windows computers.</li>
<li><a href="/wiki/IEC61850" title="IEC61850" class="mw-redirect">IEC 61850</a> a standard for electrical substation automation</li>
<li>BPA PDCStream - a variant of IEEE 1344 used by the Bonneville Power Administration (BPA) PDCs and user interface software.<sup id="cite_ref-WAMS-1_8-2" class="reference"><a href="#cite_note-WAMS-1-8"><span>[</span>8<span>]</span></a></sup></li>
</ul>
